# 更新日志

本项目的重要变更记录如下。

## [0.0.3] - 2026-03-10

### 发布者切换

- 发布者身份由 `mumu2023` 切换为 `mumuzi2023`。
- 同步更新 Marketplace 校验、Chat Participant ID 与 MCP provider ID 前缀。

## [0.0.2] - 2026-03-10

### 命名迁移与稳定性修复

- 全面迁移扩展命名空间到 `qqConnector.*`，清理旧的 `ncat.*` 配置、命令、视图与资源命名。
- 修复命令激活时机，避免安装后出现 `qqConnector.connect` 找不到的问题。
- 修复 runtime 缓存常量引用错误，解决扩展激活失败与顶部持续加载的问题。
- 调整 Activity Bar / Sidebar 图标，统一为新的 QQ 图标资源。

## [0.0.1] - 2026-03-10

### 首次对外发布

- 提供 VS Code 侧边栏 QQ 会话界面，包含会话列表、聊天详情页、搜索、会话隐藏与缓存清理。
- 默认接入 QQ 官方 Bot API，支持私聊和群聊的文本消息收发，并支持图片发送与 QQBot Markdown 文本模式。
- 注册 VS Code Chat 参与者 `@qq`，可在 VS Code Chat 中直接使用同一套模型与工具链能力。
- 提供 QQBot MCP 服务，可供 Copilot 调用，内置发送私聊消息、发送群消息、配置主对话、读取联系人、读取消息列表、读取人员列表、查看状态等工具。
- 提供本地 HTTP bridge，可把外部请求转到本地 QQ Assistant 编排链路，并返回镜像结果。
- 支持 QQ 远程消息自动回复：收到 QQ 消息后，可使用 VS Code 内的语言模型和工具自动处理并回发到 QQ。
- 支持远程工具审批：当 QQ 侧请求触发敏感工具时，会先回发确认信息，用户通过 `y/n` 继续或取消。
- 对部分本地动作工具提供确认后直通执行，包括终端命令、VS Code 命令和打开浏览器页面。
- 提供多窗口远程路由命令：`@list` 查看已注册窗口，`@path` 切换当前会话路由窗口，`@model` 查看当前目标窗口的模型列表。
- Webview UI 跟随 VS Code 主题，适配浅色/深色模式，支持图片大图悬停预览、视频悬停播放、回复引用、合并转发预览、JSON 消息发送、表情包面板等聊天增强能力。
- 针对 QQ 官方 openid 场景做了匿名化显示优化，私聊列表可显示为“用户1/用户2”，同时保留 openid 作为辅助标识。
- 保留本地兼容后端相关代码路径，但默认设置面已收敛为 QQBot 主流程，只暴露 QQBot 和安全相关配置项。
